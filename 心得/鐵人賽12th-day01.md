雖然知道前端三大重點（HTML / CSS / JavaScript）的基礎入門鐵人賽主題已相當多，仍想以自己的方式去詮釋一遍，藉此重新深化自己的開發基本功，將一些學習盲區給爬梳清楚。

大致上寫文的方式會重點式的概念陳述，不會細節到每一個步驟手把手詳列，希望能較類似於學習地圖的效用，讓前端新手讀者能抓到些關鍵字及重點去自行練習、爬文深入探究。

希望最後留存在這的系列文章能夠真正幫助到剛入坑前端的新手們，也歡迎路過的各路大大們能給予各種指教與交流，技術文若有寫錯的觀念也敬請留言指正協助勘誤，感激不盡！

2020 年初離開了待四年多的傳產公司，到某學院參加了主打轉職的全端培訓營，歷經各種心情轉折（~~懷疑人森~~），在年中如願於某軟體公司任職前端工程師。

目前工作還處於兢兢業業的菜鳥戰鬥戒備狀態。邊工作持續點擊技能樹中，時時刻刻調整自己的工作與學習心態。  
認為生活如同軟體開發一樣，是一個持續修正與優化的過程－－修心、修性、修思維、修技能。



## Eudora x 鐵人賽

《原子習慣》一書提醒了我，想達成目標，比「計畫」更重要的是「動機」－－「為何而做？為了改變什麼？為了想得到什麼？」

故首篇也來問問自己：「為什麼參加鐵人賽？」


1. 自我爬梳  
   檢視過往的程式筆記，總有些模糊地帶的學習盲區，趁著這次強迫寫作的去逐一釐清。


2. 互相 (~~傷害~~) 激勵   
   現實中認識的工程師朋友們也參賽了，一起共同成長！


3. 社群交流  
    發現技術社群交流也是一件頗為重要的事，以往習慣默默潛水的我，透過這次鐵人賽來感受一下社群氛圍囉。  
    

4. 找回網路創作的手感  
    憶起 N 年前臉上滿滿膠原蛋白、身為青春洋溢高中妹的自己，頂著咩咩頭瀏海一臉天真無邪（~~還沒遭受職場妖魔鬼怪的摧殘~~），帶著一顆單純無懼（~~無汙染~~）的心參加部落格競賽，純粹在網路上分享日常的大小事、寫寫散文、寫寫青少年的各種感性（~~為賦新詞強說愁~~），好快樂。（那是個社群媒體還沒興起、智慧型手機還沒普及的時代啊！）  

   那時的自己不太會在意他人的眼光，總能書寫自己當下內心想寫的，好自由。  
   直到失去了這個最原始的能力後才漸漸發現，不再寫是因為我「不夠勇敢」。社會化的過程變得過度在意別人的想法，也漸漸不想被看透內心的自己，所以在網路失了聲。  
   
   可是，創作明明能帶給自己很多正向的能量、也能夠與更多有故事的人交流，怎可因小失大呢？
   哎～來從不用這麼走內心戲的技術文章發表開始練習好了！
    

5. 讓我的 2020 再多一項有意義的事吧  
   2020 對於自己特別有意義，選擇離職、全時進修、轉職、及結束一段數年的關係。遇見了許多新的人、事、物，也更了解了內心深處的自己。  
   看似順利轉職成功、也得到比轉職前預期更好的工作，但冒牌者症候群的症狀卻更深了……  
   只好邊調適自己的自我認知、也同時持續做些什麼來提升自己的技術及自信，就從鐵人賽開始吧！
